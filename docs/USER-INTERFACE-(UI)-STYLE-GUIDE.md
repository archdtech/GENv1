# User Interface (UI) Style Guide

## Design Philosophy

**"Clean, Scientific, and Empowering"**

MetaBite's interface design balances scientific credibility with approachable, user-friendly interactions. The design conveys precision and trustworthiness while remaining warm and encouraging. We believe that health technology should feel both professional and human, complex data should be presented simply, and users should feel empowered rather than overwhelmed.

### Core Design Principles
1. **Clarity First:** Every element should have a clear purpose and be easily understood
2. **Data-Driven Beauty:** Visualize complex information in elegant, digestible ways
3. **Progressive Disclosure:** Reveal complexity gradually as user expertise grows
4. **Consistent Predictability:** Users should always know what to expect
5. **Emotionally Supportive:** Design should encourage and motivate health journeys

| **H4** | 16px | 600 | 1.5 | Card subtitles, small headers |

#### Body Text
| Level | Font Size | Weight | Line Height | Usage |
|-------|-----------|--------|-------------|-------|

| **Body Regular** | 14px | 400 | 1.5 | Standard body text, descriptions |
| **Body Small** | 12px | 400 | 1.4 | Supporting text, captions, footers |

#### Special Text
| Level | Font Size | Weight | Line Height | Usage |
|-------|-----------|--------|-------------|-------|
| **Button Text** | 14px | 500 | 1.2 | Button labels, interactive elements |
| **Caption** | 11px | 400 | 1.3 | Image captions, tiny text |
| **Code/Mono** | 13px | 400 | 1.4 | Data, technical information |

### Typography Guidelines
- **Hierarchy:** Use size and weight to create clear visual hierarchy
- **Readability:** Maintain proper line height (1.4-1.6) for body text
- **Consistency:** Use established sizes consistently across the application
- **Accessibility:** Never use text smaller than 11px for important information
- **Emphasis:** Use weight and color rather than underline for emphasis

## Iconography

### Icon Style
**Style:** Clean, minimalist, geometric  
**Stroke Weight:** 2px consistent stroke  
**Corner Radius:** Slightly rounded corners for approachability  
**Viewbox:** 24x24 standard grid for consistency

### Icon Categories

#### Action Icons
- **Primary Actions:** Filled style with primary color
- **Secondary Actions:** Outline style with secondary color
- **Destructive Actions:** Outline style with error red

#### Status Icons
- **Success:** Checkmark circle, filled with success green
- **Warning:** Exclamation triangle, filled with warning yellow
- **Error:** X circle, filled with error red
- **Info:** Information circle, filled with info blue

#### Navigation Icons
- **Active State:** Filled with primary color
- **Inactive State:** Outline with medium gray
- **Hover State:** Outline with primary color

#### Data Icons
- **DNA/Genetics:** Double helix, genetic strand variations
- **Nutrition:** Food items, nutritional symbols
- **Activity:** Exercise equipment, movement indicators
- **Progress:** Charts, graphs, achievement symbols

### Icon Usage Guidelines
- **Consistency:** Use established icons consistently for the same actions
- **Clarity:** Icons should be immediately recognizable without labels
- **Size:** Use 24px for standard icons, 16px for small icons, 32px for large icons
- **Spacing:** Maintain consistent padding around icons (minimum 8px)
- **Accessibility:** Always include ARIA labels for screen readers

## Imagery & Photography

### Photography Style
**Approach:** Authentic, diverse, and aspirational  
**Lighting:** Bright, natural lighting with soft shadows  
**Composition:** Clean backgrounds, focus on subjects  
**Color Tone:** Warm and inviting, slightly saturated

### Subject Matter
- **People:** Diverse age ranges, ethnicities, and body types
- **Settings:** Natural environments, home settings, gyms, kitchens
- **Activities:** Real people engaging in healthy behaviors
- **Food:** Fresh, appetizing, well-presented meals
- **Technology:** People using devices in realistic settings

### Image Guidelines
- **Authenticity:** Avoid overly staged or stock-looking photos
- **Diversity:** Represent our diverse user base
- **Relevance:** Images should support the content and message
- **Quality:** High resolution (minimum 1200px on the long edge)
- **Performance:** Optimize images for web (WebP format, compressed)

### Illustration Style
**Approach:** Friendly, scientific, and approachable  
**Style:** Flat design with subtle gradients  
**Color Palette:** Use brand colors with limited additional colors  
**Complexity:** Simple enough to understand quickly, detailed enough to be informative

### Illustration Usage
- **Concepts:** Abstract ideas like DNA, data flow, processes
- **Onboarding:** Step-by-step guides and tutorials
- **Empty States:** Friendly illustrations when no data exists
- **Education:** Explaining complex concepts simply

## Logo Usage

### Primary Logo
**Format:** Horizontal wordmark with DNA helix icon  
**Colors:** Primary Blue (#3B82F6) with Dark Gray (#111827) text  
**Clearspace:** Minimum height equal to the "M" in MetaBite on all sides

### Logo Variations

#### Horizontal Logo
- **Use:** Primary usage for headers, navigation, marketing materials
- **Minimum Size:** 120px width
- **Clearspace:** X-height on all sides

#### Stacked Logo
- **Use:** Profile pictures, app icons, square spaces
- **Minimum Size:** 80px width
- **Clearspace:** X-height on all sides

#### Icon Only
- **Use:** Favicons, app icons, small spaces
- **Minimum Size:** 32px width
- **Clearspace:** 25% of icon size on all sides

#### Monochrome Logo
- **Use:** Single-color applications, fax, engraving
- **Colors:** Black, white, or primary blue
- **Clearspace:** X-height on all sides

### Logo Usage Guidelines

#### Do's
- **Use approved logo files** from the brand asset kit
- **Maintain clearspace** around the logo
- **Ensure proper contrast** with background colors
- **Scale proportionally** to maintain aspect ratio
- **Use appropriate variation** for the context

#### Don'ts
- **Stretch or distort** the logo
- **Change colors** (except approved monochrome versions)
- **Add effects** like shadows, glows, or outlines
- **Place on busy backgrounds** that reduce readability
- **Modify typography** or letter spacing

### Logo Clearspace
- **Minimum Clearspace:** Equal to the height of the "M" in MetaBite
- **Exclusion Zone:** No other elements, text, or graphics within clearspace
- **Background Contrast:** Ensure minimum 4.5:1 contrast ratio for accessibility

## Animation & Transitions

### Animation Principles
- **Purposeful:** All animations should serve a clear purpose
- **Quick:** Animations should be fast (200-300ms)
- **Smooth:** Use easing functions for natural movement
- **Subtle:** Avoid distracting or excessive animations
- **Accessible:** Respect user's motion preferences

### Transition Types
- **Fade:** For appearing/disappearing content
- **Slide:** For drawers, menus, and mobile navigation
- **Scale:** For modals, popups, and focus states
- **Progress:** For loading states and step-by-step processes

### Easing Functions
- **Ease-in-out:** For natural movement (default)
- **Ease-out:** For quick, responsive actions
- **Ease-in:** For incoming elements
- **Linear:** For loading and progress animations

## Accessibility Guidelines

### Color Accessibility
- **Contrast Ratio:** Minimum 4.5:1 for normal text, 3:1 for large text
- **Color Blindness:** Ensure information isn't conveyed by color alone
- **Dark Mode:** Provide high contrast alternative for low-light conditions

### Interactive Elements
- **Touch Targets:** Minimum 44x44px for mobile interactions
- **Focus States:** Clear visible focus indicators for keyboard navigation
- **Error Prevention:** Confirm destructive actions, provide undo options

### Screen Reader Support
- **ARIA Labels:** Descriptive labels for interactive elements
- **Semantic HTML:** Use proper HTML elements for their intended purpose
- **Reading Order:** Logical flow that matches visual order
- **Alternative Text:** Descriptive alt text for all meaningful images

### Cognitive Accessibility
- **Simple Language:** Clear, straightforward instructions
- **Consistent Patterns:** Predictable interactions throughout
- **Error Recovery:** Clear error messages and recovery paths
- **Help Options:** Contextual help and guidance available
